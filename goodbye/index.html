UCalendar

UCalendar fue una aplicación web permitía obtener un calendario (iCal) del horario de los ramos y pruebas utilizando BuscaCursosUC, para tenerlos a mano en el calendario. Fue inicialmente creada a finales del 2020-1 como una versión de escritorio y luego pasó a una versión web disponible gratuitamente a toda la universidad.

En las últimas semanas del semestre 2022-2, BuscaCursosUC empezó a requerir iniciar sesión con SSO con una cuenta UC. Esto provocó que UCalendar dejara de funcionar, luego de cerca de 2 años de mantención. La funcionalidad de exportar las fechas de las pruebas aún no funciona luego de estos años.

UCalendar no añade carga considerable a BuscaCursos

La razón más lógica de que esta medida fue tomada, fue que plataformas como UCalendar añaden carga a BuscaCursosUC dado las consultas que se realizan para obtener información. Y sí, se ha visto plataformas que realizan consultas frecuentes y que añaden carga a BuscaCursosUC. UCalendar no es una de ellas.

UCalendar (web) fue diseñado con una base de datos intermediaria específicamente para reducir la cantidad de consultas. Una vez solicitado un curso en específico, se guarda esa información para ser reutilizada. Además, al inicio de semestre, se obtenían datos realizando consultas de cursos tipo IIC2 o MAT1 obteniendo todos los cursos de esa área.

No se registró la cantidad de consultas realizadas durante el funcionamiento (registro que se debió haber hecho), pero dado el funcionamiento de la plataforma, es seguro decir que no añadía carga considerable a BuscaCursosUC.

La medida perjudica a todos y hay mejores alternativas

Al igual que el captcha que se agregó en la plataforma de toma de ramos (para bloquear una plataforma web automatizada creada por alumnos), esta medida perjudica a los estudiantes y a las plataformas que ayudan a los estudiantes, y solo beneficia algo la carga de los servidores de la UC.

Tampoco añade justicia, dado plataformas como UCalendar han estado disponibles para todos y sin costo. Realizar estas tareas por cuenta propia y de forma automatizada siempre se ha podido (eso incluye evadir el SSO y el captcha) pero requieren un setup más complejo.

En vez de usar SSO, podrían utilizar rate-limiting, que bloquea ips (computadores) que realizan muchas consultas en un bajo periodo de tiempo y no rompería aplicaciones como UCalendar.

Otra opción, es que los datos públicos de la universidad puedan ser utilizados públicamente con una API Web. También, para casos como la toma de ramos, se podría cambiar el funcionamiento para que no sea necesario estar logueado en el segundo exacto para tomar un ramo.

Al igual que el cambio de Drive (donde se dijo que este cambio se consideró el menor impacto para la comunidad UC), creo que falta escuchar más a la comunidad UC (y ver que opina el DCC) antes de realizar este tipo de cambios. Además, la UC debería ser más transparente con estas decisiones, explicando por qué se toman, los costos y beneficios.
